# 常见问题

## 通用问题

### Q: 这个项目是做什么的？

A: 这是一个前端页面统计监控系统，类似于 Google Analytics，可以帮你监控网站的访问情况和性能指标。

### Q: 需要什么技术栈？

A:
- 后端：Node.js + Express + TypeORM + MySQL
- 前端：Vue 3 + Vite + Pinia + Element Plus
- SDK：原生 JavaScript

### Q: 可以免费使用吗？

A: 是的，这是 MIT 许可证的开源项目，可以免费使用和修改。

## 安装和配置

### Q: 如何快速开始？

A:
1. 使用 Docker Compose：`docker-compose up -d`
2. 或手动部署各服务，参考部署文档

### Q: 数据库如何初始化？

A:
- Docker 方式会自动初始化
- 手动方式执行 `scripts/init-db.sql`

### Q: 忘记管理员密码怎么办？

A:
1. 连接数据库
2. 删除或更新 users 表中的 admin 记录
3. 重新运行种子数据脚本

## 使用问题

### Q: 如何创建监控项目？

A:
1. 登录系统
2. 进入项目管理页面
3. 点击"新建项目"
4. 填写项目信息
5. 获取项目 Key

### Q: 如何在网站中集成 SDK？

A: 在需要监控的网站中添加：

```html
<script src="https://your-domain.com/sdk.js?key=YOUR_PROJECT_KEY"></script>
```

### Q: SDK 会影响网站性能吗？

A: SDK 是轻量级的，文件体积小于 10KB，使用异步加载，对性能影响很小。

### Q: 数据多久更新一次？

A: 数据实时上报，统计数据按天汇总，可以看到最近的数据。

## 技术问题

### Q: 支持哪些浏览器？

A:
- Chrome/Edge: 最新版本及前两个版本
- Firefox: 最新版本及前两个版本
- Safari: 最新版本
- 不支持 IE

### Q: 数据存储在哪里？

A: 所有数据存储在你自己的 MySQL 数据库中，你有完全的控制权。

### Q: 如何备份数据？

A:
```bash
mysqldump -u username -p web_monitoring > backup.sql
```

### Q: 如何升级系统？

A:
1. 备份数据库
2. 拉取最新代码
3. 安装依赖
4. 重启服务

## 隐私和安全

### Q: SDK 收集哪些数据？

A:
- 页面 URL 和标题
- 来源页面
- 屏幕分辨率
- 浏览器和操作系统信息
- 页面加载性能数据
- JavaScript 错误信息

不收集：
- 个人身份信息
- 密码和敏感信息
- 表单输入内容

### Q: 数据是否安全？

A:
- 所有数据存储在你自己的服务器
- 支持 HTTPS 加密传输
- 使用 JWT 进行身份验证
- 建议：启用 HTTPS、配置防火墙

### Q: 符合 GDPR 吗？

A: SDK 本身符合 GDPR 要求，但你需要根据实际情况配置隐私政策和用户同意机制。

## 故障排查

### Q: 页面访问数据为空？

A: 检查：
1. SDK 是否正确集成
2. 项目 Key 是否正确
3. 浏览器控制台是否有错误
4. 网络请求是否成功

### Q: 性能数据不准确？

A:
- 性能数据基于 Performance API
- 某些浏览器可能不支持部分指标
- 首次访问可能数据不完整

### Q: 统计数据不更新？

A:
- 检查后端服务是否正常运行
- 检查数据库连接是否正常
- 查看后端日志
